一、文件操作

（一）文件操作流程

用word操作一个文件的流程如下:

1．找到文件，双击打开
2. 读或修改
3．保存&关闭

用pythqn操作文件也差不多:

f = open('filename')  # 打开文件
f.write('我是野生程序员')  # 写操作
f.read()  # 读操作
f.close()  # 保持并关闭

&emsp;&emsp;不过有一点跟人肉操作word文档不同，就是word文档只要打开了，就即可以读、又可以修改。但Python只能以读、创建、追加3种模式中的任意一种打开文件，不能即写又读。

文件打开模式(文本模式)

1、r只读模式
2、w创建模式，若文件已存在，则覆盖旧文件
3、a追加模式，新数据会写到文件末尾

（六）文件修改

问：为什么原有数据会被覆盖呢？

&emsp;&emsp;这是硬盘的存储原理导致的，当你把⽂件存到硬盘上，就在硬盘上划了⼀块空间，存数据，等你下次打开这个⽂件 ，seek到⼀个位置，每改⼀个字，就是把原来的覆盖掉，如果要插⼊，是不可能的，因为后⾯的数据在硬盘上不会整体向后移。所以就出现 当前这个情况 ，你想插⼊，却变成了会把旧内容覆盖掉。

问：但是⼈家word, vim 都可以修改⽂件呀，你这不能修改算个什么玩意？

&emsp;&emsp;我并没说就不能修改了，你想修改当然可以，就是不要在硬盘上修改，把内容全部读到内存⾥，数据在内存⾥可以随便增删改查，修改之后，把内容再全部写回硬盘，把原来的数据全部覆盖掉。vim word等各种⽂本编辑器都是这么⼲的。

问：但是⽂件如果特别⼤，⽐如5个GB,读到内存，就⼀下⼦吃掉了5GB内存，好费资源呀，有没有更好的办法呢？

&emsp;&emsp;如果不想占内存，只能⽤另外⼀种办法啦，就是边读边改，什么意思？ 不是不能改么？是不能改原⽂件 ，但你可以打开旧⽂件 的同时，⽣成⼀个新⽂件呀，边从旧的⾥⾯⼀⾏⾏的读，边往新的⼀⾏⾏写，遇到需要修改就改了再写到新⽂件，这样，在内存⾥⼀直只存⼀⾏内容。就不占内存了。 但这样也有⼀个缺点，就是虽然不占内存 ，但是占硬盘，每次修改，都要⽣成⼀份新⽂件，虽然改完后，可以把旧的覆盖掉，但在改的过程中，还是有2份数据的。

问：还有更好的⽅式么?

&emsp;&emsp;有完没完？ 没了。

二、练习–全局文件检索替换

&emsp;&emsp;写⼀个脚本，允许⽤户按以下⽅式执⾏时，即可以对指定⽂件内容进⾏全局替换，且替换完毕后打印替换了多少处内容。

def file_replace(file_name, old_str, new_str):

    # 1、将数据导入内存
    file = open(file_name, mode='r+', encoding='utf-8')
    data = file.read()
    new_data = data.replace(old_str, new_str)

    # 2、删除硬盘数据
    file.seek(0)  # 将光标移动到文件最开始
    file.truncate()  # 截断：将光标后的数据全部删除

    # 3、写入新数据
    file.write(new_data)
    file.close()

    print(f'成功替换字符{old_str}到{new_str}，共计{data.count(old_str)}处')

    if log_name in info_dic.keys():

            # 可以输入三次密码
            while times < 3:
                log_password = input('please input your password: ').strip()

                if log_password == info_dic[log_name]['password']:
                    exit('welcome, you are logged in')
                # 如果密码不正确：打印提示
                else:
                    print(f'your password is incorrect, you have {2 - times} times left')
                    # 如果密码错误三次：锁定账户，打印提示；并按照原格式保存会数据库
                    if times == 2:
                        info_dic[log_name]['state'] = 'locked'
                        print('your account is locked')
                        # 字典转回txt
                        user_info1 = open('users', mode='w', encoding='utf-8')

                            user_info1.write(k + ' ' + info_dic[k]['password'] + ' ' + info_dic[k]['state'] + '\n')
                        user_info1.close()  # 源文件中的耿天霸状态也会被锁定

                times += 1
    else:
        print('username not registered')
        continue

（二）需求

1、程序启动后，给⽤户提供查询接⼝，允许⽤户᯿复查股票⾏情信息(⽤到循环)
2、允许⽤户通过模糊查询股票名，⽐如输⼊“啤酒”, 就把所有股票名称中包含“啤酒”的信息打印出来
3、允许按股票价格、涨跌幅、换⼿率这⼏列来筛选信息，⽐如输⼊“价格>50”则把价格⼤于50的股票都打印，输⼊“市盈率<50“，则把市盈率⼩于50的股票都打印，不⽤判断等于
&emsp;&emsp;思路提示：加载⽂件内容到内存，转成dict or list结构，然后对dict or list 进⾏查询等操作。 这样以后就不⽤每查⼀次就要打开⼀次⽂件了，效率会⾼。

（三）代码

# 1、导入用户数据
user_info = open('stock_data', encoding='utf-8')

# 2、将数据转化成字典，key为用户名，value为一个字典，包含{股票代码：，名称：，当前价：...}
info_dic = {
    # 'SH601778'：{'股票名称': 'N晶科', '当前价': '6.29', '涨跌幅': '+1.92'... },
    # 'SHxxxxxx':{'股票名称': 'xxx', '当前价': 'x.xx', '涨跌幅': 'xxxxx'... }
}

# 遍历每一行
for line in user_info:
    # print(line)
    stock = line.strip().split(',')  # 先用strip()去掉回车（'\n'），再用split()按逗号进行分割
    # print(stock)
    # print(len(stock))
    info_dic[stock[0]] = {
        '股票名称': stock[1],
        '当前价': stock[2],
        '涨跌额': stock[3],
        '涨跌幅': stock[4],
        '年初⾄今': stock[5],
        '成交量': stock[6],
        '成交额': stock[7],
        '换⼿率': stock[8],
        '市盈率(TTM)': stock[9],
        '股息率': stock[10],
        '市值': stock[11],
    }
# print(info_dic)
# 删除第一行
del info_dic['股票代码']

count = 0
while True:
    user_input = input('股票查询接口>>：')

    # 输入e以退出程序
    if user_input == 'e':
        exit('bye!')

    # 解决查询问题
    for stock in info_dic:
        # 如果查询股票信息则直接打印，支持模糊查询
        # print(info_dic[stock]['股票名称'])
        if user_input in info_dic[stock]['股票名称']:
            print(info_dic[stock])
            count += 1

       
    print(f'共找到{count}条记录')
